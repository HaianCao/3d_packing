Training Data Format
Training Data Structure

Training data contains multiple scenarios with items and bin size for packing algorithm optimization: 

{
  "training_data": [
    {
      "items": [
        {
          "id": 0,
          "request_id": 0,
          "L": 1830,
          "W": 2370,
          "H": 2470,
          "num_axis": 2,
          "quantity": 1
        }
      ],
      "bin_size": {
        "L": 9590,
        "W": 2390,
        "H": 2570
      }
    }
  ],
  "algorithm_weights": {
    "W_lifo": 1.0,
    "W_sim_l": 1.0,
    "W_sim_w": 1.0,
    "W_sim_h": 1.0,
    "W_leftover_l_ratio": 1.0,
    "W_leftover_w_ratio": 1.0,
    "W_packable_l": 1.0,
    "W_packable_w": 1.0
  },
  "training_config": {
    "num_steps": 3,
    "max_change": 0.3,
    "evaluation_metric": "efficiency"
  }
}
Key Components
Component Description Required
training_data[] Array of training scenarios Yes
items[] Items for each scenario (includes num_axis) Yes
bin_size Warehouse dimensions for scenario Yes
parameters Stack rules and LIFO order constraints Removed
algorithm_weights Algorithm weight parameters for training Optional
training_config Training parameters (steps, max_change, metric) Yes
JSON File Structure
Required JSON Format

Use this JSON structure to upload your data. The format supports quantity fields for multiple identical items: 

{
  "bin_size": {
    "L": 9590,
    "W": 2390,
    "H": 2570
  },
  "items": [
    {
      "id": 1,
      "request_id": 1,
      "L": 610.0,
      "W": 575.0,
      "H": 1005.0,
      "quantity": 16
    },
    {
      "id": 2,
      "request_id": 2,
      "L": 1065.0,
      "W": 104.0,
      "H": 605.0,
      "quantity": 40
    },
    {
      "id": 3,
      "request_id": 3,
      "L": 900.0,
      "W": 680.0,
      "H": 1870.0,
      "quantity": 5
    }
  ],
  "parameters": {
    "stack_rule": [],
    "lifo_order": []
  }
}
Field Descriptions
Field Type Description Required
bin_size.L Number Warehouse length dimension Yes
bin_size.W Number Warehouse width dimension Yes
bin_size.H Number Warehouse height dimension Yes
items[].id Number Unique identifier for the item type Yes
items[].request_id Number Request identifier (usually same as id) Optional
items[].L Number Item length dimension Yes
items[].W Number Item width dimension Yes
items[].H Number Item height dimension Yes
items[].quantity Number Number of identical items to pack Optional
parameters.stack_rule Array Stacking rules for items (advanced) Optional
parameters.lifo_order Array LIFO order constraints (advanced) Optional
